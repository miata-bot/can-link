    API Reference spect v0.1.0

    modules

     Modules

    Spect.Config

    Root level device configuration. Versioned for introspection in the future.

    Spect.EffectSlot

    Represents an effect running the device.

    Spect.Hardware

    Describes the hardware configuration. only one instance of this table will ever exist.

    Spect.HardwareChannel

    Holds the configuration of one channel. There are always two channels available, there will never be more or less.

    Spect.HardwareTrigger

    Descirbes one of the 4 available input chcannels. There will always be 4 channels, no more no less.

    Spect.Network

    Represents a radio network instance. A network is refered to by it's ID which is a 10 bit integer.

    Spect.Network.Identity

    Represents the current device's radio identity. This should be set once and never changed.

    Spect.Network.Leader

    Represents the current device's radio leader. Leaders can be changed at runtime.

    Spect.Network.Node

    Radio node

    Spect.NetworkNode

    Join between Network and Node to allow for the same node database to be used for multiple networks. This table is populated with information about a node per this network

    Spect.Section

    Section of one entire strip There are 8 of these total that can be distributed into 8 different Effects and 4 different Triggers

    Spect.SlotSection

    Join table between section and slot
